* Sessions need only be specified once
Starting a session with =:session= keeps a long-running python for all
blocks.  From a shell, I set up a virtual environment with =python -m
venv .venv= in the same directory as this org file.

#+BEGIN_SRC python :python .venv/bin/python :session
  hello = "Hello, world!"
  hello
#+END_SRC

#+RESULTS:
: Hello, world!

All of the interpreting happens in the =*Python*= buffer.

#+begin_src python :session
  import sys
  sys.executable
#+end_src

#+RESULTS:
: /home/robb/.emacs.d/.venv/bin/python

The variable we set near the top is still available:

#+begin_src python :session
  hello
#+end_src

#+RESULTS:
: Hello, world!

* Errors are silent /here/, but not in the =*Python*= running process
