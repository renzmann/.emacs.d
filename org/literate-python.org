* Sessions need only be specified once
Starting a session with =:session= keeps a long-running python for all blocks.  From a shell, I set up a virtual environment with =python -m venv .venv= in the same directory as this org file.

#+BEGIN_src python :python .venv/bin/python :session
  hello = "Hello, world!"
  hello
#+END_src

#+RESULTS:
: Hello, world!

All of the interpreting happens in the =*Python*= buffer.

#+begin_src python :session
  import sys
  sys.executable
#+end_src

#+RESULTS:
: /home/robb/.emacs.d/org/.venv/bin/python

The variable we set near the top is still available.  We can also get the results as a =drawer=
instead of the typical =:= style with a =:results drawer= header

#+begin_src python :session :results drawer
  hello
#+end_src

#+RESULTS:
:results:
Hello, world!
:end:

The =cwd= for python is the same as =pwd= above:

#+begin_src python :session
import os

os.getcwd()
#+end_src

#+RESULTS:
: /home/robb/.emacs.d/org

* Errors are silent

Below, we try a couple examples of raising an error. They occur in the
running =*Python*= buffer, instead of appearing below

#+begin_src python :session :results output

same filename is a bit Byzantine. Any way to make a QoL improvement
there?
